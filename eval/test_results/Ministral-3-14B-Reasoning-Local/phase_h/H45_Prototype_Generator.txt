The solution to the problem involves creating a `JSONValidator` class that validates JSON-like data against a given schema. The schema defines the expected structure and constraints for each field in the data. The validation process checks if each field is present (if required), if its type matches the expected type, and if it adheres to any additional constraints such as minimum and maximum values, lengths, or the number of keys in a dictionary.

The approach is structured to ensure that all fields in the schema are validated, and any errors are collected and returned. The validation process is divided into several steps for each field:

1. **Check if the field is required and present**: If a field is marked as required and is missing from the data, an error is added.
2. **Check the type of the field**: The type of the field is checked against the expected type specified in the schema. If the type does not match, an error is added.
3. **Check additional constraints based on the type**: Depending on the expected type, additional constraints such as minimum and maximum values, lengths, or the number of keys in a dictionary are checked. These constraints are only applied if the type check passes.

The solution ensures that the validation process is thorough and adheres to the schema's rules, collecting all errors without stopping at the first one. This approach is suitable because it systematically validates each field against the schema's requirements, ensuring that the data conforms to the expected structure and constraints.

```python
class JSONValidator:
    def validate(self, data, schema):
        errors = []
        for field, field_schema in schema.items():
            required = field_schema['required']
            if required and field not in data:
                errors.append(f"Field '{field}' is required but missing")
                continue
            if field in data:
                value = data[field]
                expected_type = field_schema['type']
                type_correct = True
                if expected_type == 'str':
                    if not isinstance(value, str):
                        errors.append(f"Field '{field}' must be of type str")
                        type_correct = False
                elif expected_type == 'int':
                    if type(value) is not int:
                        errors.append(f"Field '{field}' must be of type int")
                        type_correct = False
                elif expected_type == 'float':
                    if type(value) is not float:
                        errors.append(f"Field '{field}' must be of type float")
                        type_correct = False
                elif expected_type == 'bool':
                    if type(value) is not bool:
                        errors.append(f"Field '{field}' must be of type bool")
                        type_correct = False
                elif expected_type == 'list':
                    if type(value) is not list:
                        errors.append(f"Field '{field}' must be of type list")
                        type_correct = False
                elif expected_type == 'dict':
                    if type(value) is not dict:
                        errors.append(f"Field '{field}' must be of type dict")
                        type_correct = False
                else:
                    errors.append(f"Field '{field}' has unknown type '{expected_type}'")
                    type_correct = False
                if type_correct:
                    if expected_type in ['int', 'float']:
                        if 'min' in field_schema and value < field_schema['min']:
                            errors.append(f"Field '{field}' must be >= {field_schema['min']}")
                        if 'max' in field_schema and value > field_schema['max']:
                            errors.append(f"Field '{field}' must be <= {field_schema['max']}")
                    elif expected_type == 'str':
                        if 'min_length' in field_schema and len(value) < field_schema['min_length']:
                            errors.append(f"Field '{field}' must have length >= {field_schema['min_length']}")
                        if 'max_length' in field_schema and len(value) > field_schema['max_length']:
                            errors.append(f"Field '{field}' must have length <= {field_schema['max_length']}")
                    elif expected_type == 'list':
                        if 'min_length' in field_schema and len(value) < field_schema['min_length']:
                            errors.append(f"Field '{field}' must have length >= {field_schema['min_length']}")
                        if 'max_length' in field_schema and len(value) > field_schema['max_length']:
                            errors.append(f"Field '{field}' must have length <= {field_schema['max_length']}")
                    elif expected_type == 'dict':
                        if 'min_length' in field_schema and len(value) < field_schema['min_length']:
                            errors.append(f"Field '{field}' must have {field_schema['min_length']} or more keys")
                        if 'max_length' in field_schema and len(value) > field_schema['max_length']:
                            errors.append(f"Field '{field}' must have {field_schema['max_length']} or fewer keys")
        return {
            "valid": len(errors) == 0,
            "errors": errors
        }
```

The code defines a `JSONValidator` class with a `validate` method that takes `data` and `schema` as arguments. It iterates over each field in the schema, checks if the field is required and present, validates the type, and then checks additional constraints based on the type. Errors are collected and returned in a dictionary with a `valid` key indicating whether the data is valid and an `errors` key containing a list of error messages.
