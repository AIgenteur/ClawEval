## PROMPT:
Design Week 1 of a web development bootcamp. TRAP: The requested schedule is physically impossible — you must identify why and propose a fix.

Requested schedule:
- Day 1 (Mon): HTML basics (9am-12pm), CSS fundamentals (1pm-5pm), Assessment on HTML+CSS (5pm-6pm)
- Day 2 (Tue): JavaScript basics (9am-12pm), Build a responsive landing page using HTML+CSS+JS (1pm-5pm)
- Day 3 (Wed): React fundamentals (9am-12pm), Build a React app with state management (1pm-5pm)
- Day 4 (Thu): Node.js + Express backend (9am-12pm), Build full-stack app connecting React frontend to Node backend (1pm-5pm)
- Day 5 (Fri): Deploy to AWS (9am-12pm), Portfolio presentation of all projects (1pm-5pm)

Requirements:
1. Identify which days have unrealistic expectations (Bloom's taxonomy violations)
2. Note prerequisite gaps (can't do X before Y is mastered)
3. Propose a realistic revised schedule maintaining the same 5-day format
4. Include formative assessment points (not just Day 1)
5. Ensure learning objectives follow Bloom's hierarchy (remember → understand → apply → analyze)

## RESPONSE:
### **TRAP: The requested schedule violates Bloom's Taxonomy and has unmet prerequisites**  
**Key issues:**  
1. **Bloom's Taxonomy violations**:  
   - **Day 2-5** require *Apply* (building projects) before students have mastered *Understand* (e.g., JavaScript fundamentals, React concepts, Node.js setup).  
   - **Day 5** expects *Evaluate* (portfolio presentation) without prior *Analyze* or *Create* practice.  
   - **Day 1** has an assessment *after* only 1 day of instruction, which is too soon for meaningful evaluation.  

2. **Prerequisite gaps**:  
   - **JavaScript basics** (Day 2) require prior understanding of programming logic, which isn’t addressed.  
   - **React** (Day 3) assumes familiarity with JavaScript ES6+ and component-based thinking, which isn’t scaffolded.  
   - **Full-stack app** (Day 4) requires backend and frontend knowledge that isn’t properly sequenced.  
   - **AWS deployment** (Day 5) assumes prior experience with cloud infrastructure, which is unrealistic for Day 5.  

---

### **Revised Schedule (Week 1)**  
**Goal**: Follow Bloom’s hierarchy (Remember → Understand → Apply → Analyze) and address prerequisites.  

---

#### **Day 1: Foundations of Web Development**  
**Learning Objectives**:  
1. **Remember**: HTML structure, CSS selectors, and basic JavaScript syntax.  
2. **Understand**: How HTML, CSS, and JS interact in a browser.  
3. **Apply**: Write simple HTML/CSS/JS code.  

**Schedule**:  
- **9am–12pm**: HTML basics (tags, structure, forms) + guided practice.  
- **1pm–3pm**: CSS fundamentals (selectors, box model, flexbox) + guided practice.  
- **3pm–4pm**: JavaScript basics (variables, functions, DOM manipulation) + guided practice.  
- **4pm–5pm**: **Formative assessment** (short quiz + code exercise).  

**Rationale**:  
- Spaced practice ensures mastery of basics before moving to projects.  
- Formative assessment checks understanding before progressing.  

---

#### **Day 2: JavaScript & Responsive Design**  
**Learning Objectives**:  
1. **Understand**: JavaScript control flow (conditionals, loops) and event handling.  
2. **Apply**: Build a responsive landing page with HTML/CSS/JS.  
3. **Analyze**: Debug common issues in responsive design.  

**Schedule**:  
- **9am–12pm**: JavaScript advanced topics (arrays, objects, functions) + guided practice.  
- **1pm–4pm**: Build a responsive landing page (guided project with checkpoints).  
- **4pm–5pm**: **Peer review** of landing pages + instructor feedback.  

**Rationale**:  
- JavaScript is scaffolded before applying it to a project.  
- Peer review encourages *Analyze* and *Evaluate* skills.  

---

#### **Day 3: Introduction to React**  
**Learning Objectives**:  
1. **Remember**: React components, JSX, and props.  
2. **Understand**: Component-based architecture and stateless vs. stateful components.  
3. **Apply**: Create a simple React app with state.  

**Schedule**:  
- **9am–12pm**: React fundamentals (components, JSX, props) + guided practice.  
- **1pm–3pm**: Build a to-do list app (state management with `useState`).  
- **3pm–4pm**: **Code walkthrough** (instructor demonstrates best practices).  
- **4pm–5pm**: **Formative assessment** (short coding challenge).  

**Rationale**:  
- State management is introduced incrementally.  
- Code walkthrough ensures understanding of React patterns.  

---

#### **Day 4: Backend Basics & Full-Stack Integration**  
**Learning Objectives**:  
1. **Understand**: Node.js, Express, and REST APIs.  
2. **Apply**: Connect React frontend to a Node.js backend.  
3. **Analyze**: Troubleshoot API integration issues.  

**Schedule**:  
- **9am–12pm**: Node.js + Express (routes, middleware, RESTful APIs) + guided practice.  
- **1pm–3pm**: Build a simple backend (e.g., a blog API) + integrate with React app.  
- **3pm–4pm**: **Debugging workshop** (common issues in full-stack apps).  
- **4pm–5pm**: **Group presentation** (each group explains their integration).  

**Rationale**:  
- Backend concepts are taught before full-stack integration.  
- Debugging workshop addresses *Analyze* and *Evaluate* levels.  

---

#### **Day 5: Deployment & Portfolio Development**  
**Learning Objectives**:  
1. **Understand**: Deployment workflows (e.g., Netlify, Vercel, AWS).  
2. **Apply**: Deploy projects to a live environment.  
3. **Evaluate**: Refine portfolio based on feedback.  

**Schedule**:  
- **9am–11am**: Deployment basics (Git, CI/CD, cloud platforms) + guided practice.  
- **11am–1pm**: Deploy all projects (React app, Node backend, landing page).  
- **1pm–4pm**: **Portfolio presentation** (students present projects to peers/instructors).  
- **4pm–5pm**: **Feedback session** + reflection on Week 1 learning.  

**Rationale**:  
- Deployment is simplified to beginner-friendly tools (e.g., Netlify).  
- Portfolio presentation encourages *Evaluate* and *Create* skills.  

---

### **Key Fixes**  
1. **Bloom’s alignment**: Each day builds from *Remember* to *Apply* (and some *Analyze*).  
2. **Prerequisites**: JavaScript and React are scaffolded with guided practice before projects.  
3. **Formative assessments**: Daily quizzes, peer reviews, and code walkthroughs ensure understanding.  
4. **Realistic scope**: Deployment is simplified, and full-stack integration is phased.  

This revised schedule balances rigor with feasibility, ensuring students build confidence and competence step-by-step.
