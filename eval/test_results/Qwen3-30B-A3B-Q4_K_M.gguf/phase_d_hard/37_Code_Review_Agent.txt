#### Why is this a problem?

When multiple threads call `transfer` on the same account, the code can **race to modify the balance**. For example:

- Thread 1 checks `self.balance >= amount` and proceeds.
- Thread 2 also checks `self.balance >= amount` and proceeds.
- Both threads then subtract the same amount from `self.balance`, leading to **overdraw** and **money loss**.

This is a classic **race condition**, and the lack of locking is the root cause.
