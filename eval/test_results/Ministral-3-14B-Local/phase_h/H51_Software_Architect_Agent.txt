### Key Trade-offs:
1. **OT vs. CRDT**: OT is simpler for text but requires server coordination; CRDTs are conflict-free but have higher memory overhead.
2. **WebSocket vs. SSE**: WebSocket is bidirectional (needed for collaborative edits), while SSE is unidirectional (sufficient for notifications).
3. **MongoDB vs. PostgreSQL**: MongoDB’s flexibility wins for nested document structures, but PostgreSQL could be considered for complex queries (e.g., full-text search).
4. **Offline Handling**: PouchDB adds complexity but ensures seamless offline support; pure IndexedDB requires custom conflict resolution.
