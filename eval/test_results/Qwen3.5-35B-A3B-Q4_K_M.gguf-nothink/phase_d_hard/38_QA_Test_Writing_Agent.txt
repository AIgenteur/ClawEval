Write tests for this pagination function. It has a SUBTLE BUG — your tests must catch it. 

```python
def paginate(items, page, per_page=10):
    """Return a page of items with metadata."""
    total = len(items)
    total_pages = total // per_page + (1 if total % per_page else 0)
    
    start = (page - 1) * per_page
    end = start + per_page
    
    return {
        "items": items[start:end],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
```

## The Bug

**The bug is in the `start` calculation:** `start = (page - 1) * per_page`

When `page=0` or `page` is negative:
- `page=0` → `start = (0-1) * 10 = -10` → `items[-10:0]` returns the last 10 items (wrong!)
- `page=-1` → `start = (-1-1) * 10 = -20` → `items[-20:0]` returns the last 20 items (wrong!)

**Expected behavior:** Pages should be 1-indexed (page 1, 2, 3...), and invalid pages (0, negative, or beyond range) should return empty items.

**Fix:** Add validation to ensure page >= 1, or handle negative/zero pages by returning empty items:

```python
def paginate(items, page, per_page=10):
    if page < 1:
        page = 1  # or return empty items
    
    total = len(items)
    total_pages = total // per_page + (1 if total % per_page else 0)
    
    start = (page - 1) * per_page
    end = start + per_page
    
    # Ensure start doesn't go negative
    if start < 0:
        start = 0
    
    return {
        "items": items[start:end],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
```

The test `test_negative_page()` will fail because it expects empty items but gets the last 10 items instead.
