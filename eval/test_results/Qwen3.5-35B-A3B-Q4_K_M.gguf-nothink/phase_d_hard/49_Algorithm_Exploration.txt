## PROMPT:
Design an algorithm for this problem. There's an O(n) solution but most people give O(n log n) or O(n²). Find the optimal one.

Problem: Given an array of n integers, find the maximum difference arr[j] - arr[i] such that j > i (the larger element must come AFTER the smaller element). If no positive difference exists, return 0.

Example: [2, 3, 10, 6, 4, 8, 1] → Answer: 8 (10-2)
Example: [7, 9, 5, 6, 3, 2] → Answer: 2 (9-7, not 5-2 because 5 comes after 9)
Example: [10, 8, 6, 4, 2] → Answer: 0 (strictly decreasing)

Requirements:
1. Give the naive O(n²) solution and explain why it's slow
2. Give an O(n log n) solution using a common technique
3. Give the OPTIMAL O(n) solution — track minimum so far
4. Prove the O(n) solution is correct with a formal argument
5. Handle edge cases: empty array, single element, all same values, all negative
6. Provide Python implementation of the O(n) solution with tests

## RESPONSE:
## 1. Naive O(n²) Solution

**Approach:**
Check every possible pair (i, j) where j > i and compute arr[j] - arr[i]. Track the maximum difference found.

```python
def max_diff_naive(arr):
    if len(arr) < 2:
        return 0
    
    max_diff = 0
    n = len(arr)
    
    for i in range(n):
        for j in range(i + 1, n):
            diff = arr[j] - arr[i]
            if diff > max_diff:
                max_diff = diff
    
    return max_diff
```

**Why it's slow:**
- Time complexity: O(n²) because we have nested loops checking all pairs
- Space complexity: O(1)
- For n = 10,000, this performs ~50 million operations, which is impractical for large inputs

## 2. O(n log n) Solution Using Sorting

**Approach:**
Use a modified merge sort or track minimum elements. A common technique is to use a sorted structure or divide-and-conquer.

**Divide-and-Conquer Approach:**
Split the array into two halves. The maximum difference is either:
1. Entirely in the left half
2. Entirely in the right half  
3. Crossing the midpoint (min from left, max from right)

```python
def max_diff_divide_conquer(arr):
    if len(arr) < 2:
        return 0
    
    def helper(arr, left, right):
        if left >= right:
            return 0
        
        mid = (left + right) // 2
        max_diff = max(helper(arr, left, mid), helper(arr, mid + 1, right))
        
        # Find min in left half and max in right half
        min_left = min(arr[left:mid+1])
        max_right = max(arr[mid+1:right+1])
        
        max_diff = max(max_diff, max_right - min_left)
        
        return max_diff
    
    return max(0, helper(arr, 0, len(arr) - 1))
```

**Time Complexity:** O(n log n) due to the recurrence T(n) = 2T(n/2) + O(n)

## 3. Optimal O(n) Solution

**Approach:**
Track the minimum element seen so far as we iterate through the array. For each element, calculate the difference with the minimum seen so far and update the maximum difference.

```python
def max_diff_optimal(arr):
    if len(arr) < 2:
        return 0
    
    min_so_far = arr[0]
    max_diff = 0
    
    for i in range(1, len(arr)):
        # Calculate difference with minimum seen so far
        diff = arr[i] - min_so_far
        
        # Update maximum difference if current difference is larger
        if diff > max_diff:
            max_diff = diff
        
        # Update minimum so far if current element is smaller
        if arr[i] < min_so_far:
            min_so_far = arr[i]
    
    return max_diff
```

## 4. Formal Proof of O(n) Correctness

**Theorem:** The algorithm correctly finds max(arr[j] - arr[i]) where j > i.

**Proof:**

Let the optimal solution be arr[j*] - arr[i*] where j* > i* and this difference is maximum.

**Key Insight:** At any position j, the maximum difference ending at j is arr[j] - min(arr[0], arr[1], ..., arr[j-1]).

**Algorithm maintains invariant:** At iteration i (processing arr[i]), `min_so_far` = min(arr[0], arr[1], ..., arr[i]).

**Proof by contradiction:**
1. Suppose the algorithm returns a value less than the optimal difference.
2. Let the optimal pair be (i*, j*) where j* > i*.
3. When the algorithm processes index j*, `min_so_far` must be ≤ arr[i*] (since i* < j* and we've seen arr[i*]).
4. Therefore, the algorithm computes arr[j*] - min_so_far ≥ arr[j*] - arr[i*].
5. Since max_diff tracks the maximum of all such differences, max_diff ≥ arr[j*] - arr[i*].
6. But max_diff cannot exceed the optimal difference by definition.
7. Therefore, max_diff = arr[j*] - arr[i*].

**Conclusion:** The algorithm finds the optimal solution.

## 5. Edge Cases Handling

The algorithm handles all edge cases:
- **Empty array:** Returns 0 (handled by length check)
- **Single element:** Returns 0 (handled by length check)
- **All same values:** Returns 0 (differences are all 0)
- **All negative:** Returns 0 or positive if there's any increase
- **Strictly decreasing:** Returns 0 (no positive differences)
- **All positive:** Works correctly

def max_diff_optimal(arr):
    """
    Find the maximum difference arr[j] - arr[i] where j > i.
    Time Complexity: O(n)
    Space Complexity: O(1)
    """
    if len(arr) < 2:
        return 0
    
    min_so_far = arr[0]
    max_diff = 0
    
    for i in range(1, len(arr)):
        diff = arr[i] - min_so_far
        if diff > max_diff:
            max_diff = diff
        if arr[i] < min_so_far:
            min_so_far = arr[i]
    
    return max_diff
